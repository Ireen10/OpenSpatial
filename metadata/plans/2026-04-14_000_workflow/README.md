## Metadata 子项目工作流约定（必须遵守）

本目录用于管理我在 `metadata/` 子项目上的所有设计、计划、测试与变更记录文档。

### 总流程（任何实现之前）

对每一个功能/变更，严格按以下顺序推进，且**每一步都必须先有成文文档**并与你对齐确认后才能进入下一步：

1. **方案设计**（Design）
2. **执行计划**（Plan）
3. **测试方案**（Test Plan）
4. **代码实现**（Implementation）

### 文档要求

- **必须落盘**：每一步的文档都要提交到仓库（放在 `metadata/plans/<timestamp>_<topic>/`）。
- **时间戳命名**：目录名必须带时间戳，格式：
  - `YYYY-MM-DD_HHMM_<topic>`（例如 `2026-04-14_1530_metadata_ingest`）
- **变更留痕**：每次功能新增/变更都要新建一个新的时间戳目录，并在其中包含：
  - `design.md`
  - `plan.md`
  - `test_plan.md`
  - `change_log.md`（描述本次与上一版相比改了什么）

### 本目录结构

- `templates/`：文档模板（复制一份到新的时间戳目录再填写）
- `2026-04-14_000_workflow/`：当前这份工作流约定（基线）

本文档用于指导在 OpenSpatial 中新增「2D 空间关系」annotation task 时，如何从上游数据构建可标注/可训练的 QA，并定义质量目标与质检流程。

## 1. 任务定义

### 1.1 任务目标

在物体定位能力的基础上，识别 2D 图像平面中两个对象之间的相对方位关系（包含上、下、左、右、左上、左下、右上、右下的平面八向方位识别）。

任务约束：
不考虑真实 3D 空间关系与相机透视效应；该任务要求模型识别的是图像平面内的 2D 空间关系。

### 1.2 输入与输出形式

本节定义该任务对**模型输入（Input）**与**模型输出（Output）**的形式约定，即：输入给模型什么信息，模型预期输出什么。

#### 模型输入（Input）

输入由**单张 RGB 图片**与**一段文本**组成：

- 图片（单视角）
  - 形式：单张 RGB 图片。
  - 标注形态：可为**无标记原图**，也可为**带 2D 框标记的图**。
- 文本
  - 必须包含对 **anchor** 与 **target** 的指代表达（referring expression）。
  - 当图片为“带框标记”时，文本中的指代需增加与标记对应的备注（例如"(the {category} in the red box)"），以显式绑定语言与视觉标记。

#### 模型输出（Output）

输出为对 **2D 图像平面内**空间方位关系的描述，覆盖：
- 基本方位：上 / 下 / 左 / 右。
- 复合方位：左上 / 左下 / 右上 / 右下。

输出形态规定为**自然语言方位描述**（允许同义表述与复合方位表述）。

规范样例：待补充。

### 1.3 目标评测集与评测指标

本任务采用 [CV-Bench](https://huggingface.co/datasets/nyu-visionx/CV-Bench) 作为目标评测基准，使用其中 **2D split** 的 **Relation** 类问题作为主评测集。

#### 评测集（Evaluation Set）

- **评测集**：CV-Bench（2D split / Relation）。
- **问题形式**：二选一（Binary choice）的相对方位判断，例如询问 *A 相对 B* 是“左边还是右边”，或“上边还是下边”等。
- **评测指标**：Accuracy（Acc）。

## 2. 目标 QA 形式（要产哪几种 QA，以及依据）

### 2.1 问法清单（面向生产的目标形式）

本任务的“目标 QA 形式”指**问法/交互形式**（而非任务类型）。当前规划支持三种问法：

- 问法 1：**自然语言方位描述（八向）**
  - Question：包含任务描述（以 2D 图像平面为准、要求输出方位描述）+ 要求判断 anchor 与 target 相对位置关系的语句。
  - Answer：用**完整句子**输出**平面八向方位**（上/下/左/右/左上/左下/右上/右下），例如：
    - “从图像中可见，{target} 在图像平面上相对于 {anchor} 位于{方位}。”

- 问法 2：**单轴选项题（与 CV-Bench 2D Relation 风格一致）**
  - Question：询问 {target} 相对 {anchor} 的位置，并提供**单一维度**的候选项（例如仅 left/right 或仅 above/below），避免复合方位带来的困扰。
  - Answer：从给定选项中选择（如 “A” 或 “left”，以具体模板约定为准）。

- 问法 3：**判断题（True/False + 纠错）**
  - Question：给出一个方位断言（如“考虑 {target} 和 {anchor} 在图像平面上的位置, {target} 位于 {anchor} 的左侧吗？”）并明确要求模型输出格式。
  - Answer：
    - 若断言正确：输出“正确”，或“正确 + 完整复述方位关系”（以题面要求为准）。
    - 若断言部分正确（例如真实为“左上”，题面仅问“左”）：输出“正确，但是更准确的描述是……”，并给出更精确的八向方位描述。
    - 若断言错误：必须输出“错误”并给出正确方位描述（以题面要求为准）。

### 2.2 方位标签的计算依据：

- **输入**：anchor 与 target 的 2D 边界框 `bboxes_2d`（像素坐标系）。
- **核心依据**：以两框中心点的相对位移 \((dx, dy)\) 推导方位：
  - 横向：当 \(dx \le -\tau_u\) 判为“左”，当 \(dx \ge \tau_u\) 判为“右”；\(|dx| < \tau_u\) 视为横向不可靠/近似持平
  - 纵向：当 \(dy \le -\tau_v\) 判为“上”，当 \(dy \ge \tau_v\) 判为“下”（以图像坐标系 y 向下为正）；\(|dy| < \tau_v\) 视为纵向不可靠/近似持平
  - 八向：横向与纵向组合得到“左上 / 左下 / 右上 / 右下”，若仅单轴有效则为“左/右/上/下”
- **单轴问法**：当关系为复合方位时，优先按 \(|dx|\) 与 \(|dy|\) 选择更显著维度；当两轴接近时会按权重随机选择轴向（避免固定模板偏置）；不输出复合方位。

## 3. 数据质量目标

### 3.1 基础元数据质量（2D 空间关系 metadata）

2D 空间关系 metadata 可由两类入口得到：

- 从现有 Grounding 数据（Grounding QA 与 Grounded Caption）经 adapter + enrich 计算得到；
- 或直接从已有 metadata 输入继续执行 ensure_qa / export_training 流程。

基础质量标准：

- 完整性（对象与关系覆盖）
  - Grounding 信息抽取（当上游为 grounding 时）：每条样本需能抽取到实例指代表达与对应 `bbox`（以及必要对齐信息），否则无法计算关系。
  - bbox 过滤：在关系计算前会对 objects/boxes 进行过滤（例如几何非法、越界、面积过小、极端长宽比、上限截断等）；被过滤对象需在 metadata 中留痕（含过滤原因），以保证对象集合可追溯。
  - 关系层：关系集合应覆盖“当前参与计算的对象对”。对于“已存在 image_plane 关系的对象对”可跳过重算；其余对象对若被过滤也需留痕。

- 准确性（关系正确）
  - 保留下来的关系需与 `bboxes_2d` 的几何依据一致；同一对对象在同一规则下不应出现自相矛盾的关系标注。

- 可校验性（证据与过滤依据齐备）
  - 保留关系：应保留关系计算依据（例如用于判定的 \((dx, dy)\)、阈值、以及相关中间量）。
  - 过滤关系：应保留过滤依据（例如 IoU/包含/近似持平/极小框等过滤原因及对应阈值），以支持离线复核与分析。

### 3.2 指代描述与 Grounding 质量

当 metadata 来源于 Grounding 数据（Grounding QA 与 Grounded Caption）时，上游数据的指代表达质量与定位质量标准，直接作为 2D 空间关系 metadata 中指代表达与 grounding 质量标准。

- 指代不包含位置信息：指代表达中不应出现图像平面方位/位置线索（如“左边/右侧/上方/左上角/画面右边”等），避免将空间关系答案信息泄露到输入文本中。
- 刷新结果约束：刷新后的 `phrase` 应满足“非位置词、实例可区分、与 bbox 对齐可追溯”；无法稳定生成时可按策略保留原表达或丢弃对象。

### 3.3 QA 质量

Prompt 规范：
- 任务指向明确：必须清楚表达“方位判断以**图像平面（2D）**为准”，避免与真实 3D 空间方位理解混淆。
- 语言自然：应符合自然语言表达习惯，避免模板痕迹过重。
- 多样性：为避免模型过拟合某一种固定问法，应保证提问句式的多样性。
- 不泄露答案：Prompt 不应包含可直接推出答案的方位信息或“提示性关系”（例如“位于桌子左边的椅子在桌子的什么方位”）。

Response 规范：
- 方位描述准确：输出的方位应与图像平面中的相对位置一致。
- 指令遵从：严格按题面要求的输出格式作答（完整句/二选一/判断+纠错等）。
- 参照系声明：回答中必须明确该方位描述指的是**图像平面（2D）**中的方位关系。

## 4. 数据生产流程

本节描述从上游数据到最终 QA 的端到端流程，包含关键中间产物与每一步的失败处理策略。

### 4.1 端到端流程概览

- 数据入口：Grounding 数据（现有 Grounding QA 与 Grounded Caption）；
- 指代表达刷新：基于图像 + bbox 调用视觉 LLM（常用 `Qwen3-VL-32B-Instruct`）刷新 `phrase/category`；必要时先做 bbox 去重，再做刷新后去重。
- bbox 有效性筛选：对 objects/boxes 做基础过滤（几何非法、越界、面积过小、异常长宽比、上限截断），仅保留有效对象进入后续关系计算。
- 筛选对象对：对保留对象两两成对生成候选对，并按规则跳过已存在边、过滤不可靠候选对并留痕。
- 计算 2D 方位：基于两对象几何（中心点差值 \((dx,dy)\) 与阈值）推导平面方位。
- 生成 2D 空间关系 metadata：将保留关系写入 `relations`，将对象过滤与候选对过滤原因/统计写入 `aux`，形成可追溯 2D 关系元数据。
- QA 对生成：基于 metadata（对象、关系、证据）按既定问法模板生成 `qa_items`。
- 转换训练数据格式：按 pipeline 将 `metadata` 打包为pangu_ml训练数据格式（`images/*.tar + jsonl/*.jsonl`）。
- 数据质检：对 metadata、QA 与导出产物做质量检查。

### 4.2 bbox 有效性筛选逻辑

关系计算前先对 objects/boxes 进行过滤，仅保留满足几何与范围约束的对象进入后续步骤。典型规则包括：

- bbox 几何合法：`x2 > x1` 且 `y2 > y1`，且坐标非 NaN。
- point 合法：若对象使用 point 几何，需满足二维点格式与数值合法性。
- 极小框过滤：bbox 面积需不小于阈值。
- 异常长宽比过滤：bbox 的 `max(w/h, h/w)` 不超过阈值。
- 越界过滤：坐标需落在当前 `coord_scale` 的有效范围内。
- 数量上限截断：若对象数超过上限 `max_objects_per_sample`，按面积优先保留并对被截断对象留痕。

被过滤对象需要在 metadata 中留痕（包含过滤原因）。

### 4.3 对象对筛选逻辑

对保留对象集合两两成对构造候选对象对，并按规则跳过/过滤不可靠候选对：

- 跳过已存在边：若已存在同一有向 `(anchor_id, target_id)` 且 `ref_frame="image_plane"` 的关系，则不重复计算（避免重复边）。
- 候选对过滤与留痕：对重叠度过高、中心距离过近、存在明显包含关系、或两轴位移均过小的候选对进行过滤，并在 metadata 中记录过滤原因与必要证据。

### 4.4 2D 方位计算逻辑

对通过候选对筛选的对象对，使用代表点（bbox 中心或 point）计算 `delta_uv = target - anchor`，并按阈值判断横向/纵向方位：

- 当横纵两轴均显著时，输出复合方位（通过 `components` 表达），并在关系证据中保留代表点与 `delta_uv` 等计算依据。
- 当仅单轴显著时，输出单轴方位（左/右/上/下）；近似持平或不可靠候选对会被过滤并留痕。

### 4.5 QA 多样化方案

#### 4.5.1 多样化目标

主要目标：

- 覆盖不同输出能力：同时覆盖自然语言方位描述、受限选项判断、以及关系断言校正三类输出能力。
- 降低模板过拟合风险：避免模型仅学习固定问答模式，而忽略真实的空间关系理解与推理。
- 兼顾训练与评测：开放式问法用于训练模型输出自然语言空间描述；受限问法用于贴近目标评测集的题面形式；判断题用于增强模型对细粒度关系差异与错误纠正的能力。
- 支持不同难度层次：从低自由度的单轴判断，到高自由度的完整方位描述，再到带纠错要求的判断题，形成由浅入深的数据层次。

三类 QA 分别覆盖的考察维度：

- 完整句方位描述：侧重考察模型对空间关系的自然语言表达能力，以及对复合方位的整体表述能力。
- 单轴选项判断：侧重考察模型在受限输出空间下的单维度关系判别能力，并与目标评测形式保持一致。
- 关系判断与纠正：侧重考察模型对关系断言的辨析、细粒度差异识别以及错误纠正能力。
